  **Telegram-бот на базе ИИ для поиска фильмов**

  Автоматизированный бот, созданный для поиска подробной информации о фильмах, мультфильмах и сериалах с использованием искусственного интеллекта. Работает на базе новейших нейросетей Google Gemini. Бот умеет искать кино по тексту, ссылкам на YouTube, голосовым сообщениям, видеокружочкам и даже скриншотам\!

  Имеет умную ротацию API-ключей, встроенное меню настроек, поддержку прокси и легкую кастомизацию через конфигурационные файлы.

  -----

  ## ✨ Особенности

    * **Мультимодальный поиск (НОВОЕ):** Бот анализирует не только текст. Отправьте ему картинку (кадр из фильма), голосовое сообщение с напевом саундтрека, видео-кружочек с фрагментом диалога — Gemini распознает медиафайлы и найдет нужное кино.
    * **Глубокий парсинг YouTube + Cookies:** Автоматически извлекает название видео, описание и субтитры. Поддерживает загрузку файла `cookies.txt` прямо через Telegram для обхода блокировок YouTube и доступа к видео с ограничениями.
    * **Встроенное меню настроек (НОВОЕ):** Команды `/start` или `/settings` открывают удобное Inline-меню. Прямо в чате можно управлять прокси, загружать cookies и запускать умный тест API-ключей.
    * **Умная ротация ключей и прокси (НОВОЕ):** Поддерживает список API-ключей. Если ключ ловит лимит, бот переключается на следующий. Если все ключи временно заблокированы, бот автоматически активирует режим "Последний шанс" и пускает запросы через прокси.
    * **Мультиязычность (Локализация):** Все тексты кнопок, ответов и меню вынесены в файл `language.json`. Меняйте язык бота без вмешательства в код.
    * **Обработка ИИ с Google Search:** Использует встроенную интеграцию с Google Search. Нейросеть в реальном времени гуглит неочевидные детали прямо во время анализа.
    * **Вайтлист чатов:** Ограничивает ответы бота только разрешенными ID групп и конкретных веток (топиков) для предотвращения спама.
    * **Поддержка Docker:** Простой, быстрый и чистый деплой одной командой с сохранением локальных данных (прокси, куки).

  ## 🧰 Стек технологий

    * **Язык:** Python 3.11
    * **Фреймворк:** aiogram 3.x
    * **ИИ:** Google GenAI SDK (Flash / Flash Lite)
    * **Парсинг:** yt-dlp, requests
    * **Деплой:** Docker & Docker Compose

  -----

  ## 🤖 Как это работает

  1.  **Запрос:** Пользователь отправляет текст, ссылку, картинку, войс или кружочек.
  2.  **Сбор данных:** Если обнаружена ссылка на YouTube, бот с помощью `yt-dlp` (и ваших cookies) вытягивает метаданные и субтитры.
  3.  **Анализ нейросетью:** Собранный текст или медиафайл отправляется в Gemini. Происходит многоэтапный поиск (включая обращения к Google Search, если необходимо).
  4.  **Выдача результата:** Бот возвращает красиво оформленное сообщение (название, год, жанр, завязка сюжета), генерирует кнопку для трейлера и ссылку на источник. Исходное сообщение пользователя при этом удаляется для чистоты чата.

  ## 🔑 Как получить Google API Key

  1.  Перейди в [Google AI Studio](https://aistudio.google.com).
  2.  Войди с помощью своего Google аккаунта и нажми **"Get API key"**.
  3.  Сгенерируй и скопируй свой ключ. *(Совет: Настоятельно рекомендуется создать 3-4 ключа с разных аккаунтов и указать их через запятую в конфиге для стабильной работы).*

  ## 🚀 Установка и запуск

  **1. Клонирование репозитория**

  ```bash
  git clone https://github.com/newfpv/AiMovieSearcher.git
  cd AiMovieSearcher
  ```

  **2. Настройка окружения**
  Переименуй файл `.env.example` в `.env` и заполни своими актуальными данными:

  ```bash
  cp .env.example .env
  nano .env
  ```

  Укажи свои `TG_BOT_TOKEN`, `API_KEYS` (через запятую) и настройте `GROUP_THREAD_MAP` при необходимости.

  **3. Запуск бота через Docker**

  ```bash
  docker-compose up -d --build
  ```

  ## 🔄 Обновление со старой версии (Update Guide)

  В новой версии бота произошло глобальное обновление: добавлена поддержка медиафайлов, система мультиязычности (`language.json`), встроенное меню настроек и защита от блокировок YouTube.

  Чтобы обновиться без ошибок и падений, чётко следуйте этой инструкции:

  **Шаг 1. Загрузите свежий код**

  ```bash
  git pull origin main
  ```

  *(Или просто замените старые файлы на новые, если скачивали архив).*

  **Шаг 2. Обновите конфигурацию (.env)**
  Логика переменных окружения изменилась:

    * **Удалены:** `TRAILER_BTN_TEXT` и `SOURCE_BTN_TEXT`. Весь текст настраивается через `language.json`.
    * **Добавлено:** Переменная `LANG_FILE=language.json` для выбора языка.
    * **Обновлен промпт:** В `SYSTEM_PROMPT` добавлены инструкции для работы с изображениями. Обязательно обновите свой промпт, скопировав актуальный из файла `.env.example`.

  **Шаг 3. Проверьте docker-compose.yml**
  Бот теперь сохраняет настройки прокси и cookies в папку `data`. Убедитесь, что в вашем файле `docker-compose.yml` есть этот блок:

  ```yaml
      volumes:
        - ./data:/app/data
        - ./:/app
  ```

  **Шаг 4. Пересоберите Docker-контейнер**
  У бота полностью обновился список зависимостей (новый `google-genai`). **Обязательно** нужно пересобрать образ с нуля:

  ```bash
  docker-compose down
  docker-compose build --no-cache
  docker-compose up -d
  ```

  Отправьте команду `/start` или `/settings` в чат, чтобы открыть новую панель управления.

  ## ⚙️ Решение проблем

    * **Ошибки API (429 RESOURCE\_EXHAUSTED):** Вы исчерпали дневные лимиты. Добавьте больше ключей в `API_KEYS`. Откройте меню `/settings` -\> "Тест API", чтобы проверить, какие ключи сейчас живы.
    * **YouTube не отдает субтитры или видео:** Google начал блокировать запросы серверов. Откройте меню `/settings` -\> "Настройки Cookies YouTube" и загрузите файл `cookies.txt` от вашего аккаунта (инструкция по получению есть внутри самого бота).
    * **Ключи отлетают в бан по IP:** Откройте `/settings` -\> "Настройки Прокси" и добавьте рабочий HTTP/SOCKS прокси. Бот будет использовать его, когда локальные запросы начнут блокироваться.
