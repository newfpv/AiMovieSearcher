Автоматизированный бот, созданный для поиска подробной информации о фильмах, мультфильмах и сериалах с использованием искусственного интеллекта. Работает на базе новейших нейросетей Google Gemini, имеет умный парсинг ссылок YouTube, автоматическую ротацию API-ключей для обхода лимитов и гибкую настройку через файл `.env`.

## ✨ Особенности
- **Глубокий парсинг YouTube:** Бот не просто смотрит на ссылку. Он автоматически извлекает **название видео, полное описание и все доступные субтитры** (с помощью `yt-dlp` и `youtube-transcript-api`). Это позволяет безошибочно находить фильмы даже по коротким нарезкам Shorts или TikTok-эдитам без названия.
- **Обработка ИИ с Google Search:** Использует встроенную в Gemini интеграцию с Google Search. Нейросеть в реальном времени гуглит неочевидные цитаты, имена персонажей или детали сюжета прямо во время анализа.
- **Умная ротация ключей и моделей:** Поддерживает неограниченный список API-ключей. Если ключ или модель ловит лимит (ошибка 429), бот моментально и без падений переключается на следующий доступный вариант.
- **Универсальный поиск:** Ищет не только по ссылкам на видео, но и по обычному тексту. Можно просто написать примерный сюжет сцены или запомнившуюся фразу героя.
- **Авто-очистка чата:** Бот автоматически удаляет исходное сообщение пользователя (со ссылкой) после успешной выдачи результата, сохраняя идеальную чистоту в группе.
- **Полная асинхронность:** Написан на современном фреймворке `aiogram 3`. Бот не зависает и легко обрабатывает одновременные запросы от множества пользователей.
- **Надежная защита от ошибок форматирования:** Если ИИ ошибается и выдает сломанный HTML-код, бот автоматически очищает текст от тегов и отправляет его в простом виде, чтобы избежать ошибки `TelegramBadRequest`.
- **Простая локализация:** Настраивай текст кнопок интерфейса («Трейлер на YouTube», «Источник») прямо в конфигурационном файле `.env` без вмешательства в код.
- **Вайтлист чатов:** Жестко ограничивает ответы бота только разрешенными ID групп и конкретных веток (топиков) для предотвращения спама и нецелевого использования.
- **Поддержка Docker:** Простой, быстрый и чистый деплой одной командой через `docker-compose`.

* **Деплой:** Docker & Docker Compose

## 🤖 Как это работает
1. **Запрос:** Пользователь отправляет в чат ссылку на YouTube / Shorts (или просто текстовое описание фильма). Бот сразу показывает статус «печатает...».
2. **Сбор данных:** Если обнаружена ссылка, бот подключает парсеры. `yt-dlp` мгновенно вытягивает оригинальное название и описание ролика, а `youtube-transcript-api` скачивает текст субтитров (даже если они сгенерированы автоматически).
3. **Формирование контекста:** Бот склеивает текст пользователя, заголовок, описание и до 5000 символов субтитров в единый массив данных.
4. **Анализ нейросетью:** Весь собранный текст отправляется в Gemini. Нейросеть читает диалоги из субтитров, при необходимости делает скрытые запросы в Google Search для проверки фактов и определяет точное название фильма.
5. **Выдача результата:** Бот возвращает красиво оформленное сообщение (название, год, жанр, завязка сюжета), автоматически генерирует кнопку для поиска трейлера и ссылку на оригинальное видео. Исходное сообщение пользователя при этом удаляется.

## 🔑 Как получить Google API Key
Google предоставляет бесплатные API-ключи для своих нейросетей с дневными ограничениями.

**Инструкция:**
1. Перейди в [Google AI Studio](https://aistudio.google.com).
2. Войди с помощью своего Google аккаунта.
3. Нажми **"Get API key"** в левом меню.
4. Сгенерируй и скопируй свой ключ. *(Совет: Настоятельно рекомендуется сгенерировать 3-4 ключа с разных аккаунтов и указать их через запятую в конфиге для стабильной работы бота).*

### 📊 Бесплатные лимиты моделей:
- **Gemini 3 Flash:** 20 запросов / день
- **Gemini 2.5 Flash:** 20 запросов / день
- **Gemini 3.1 Flash Lite:** 500 запросов / день
- **Gemini 2.5 Flash Lite:** 20 запросов / день

## 🚀 Установка и запуск

**1. Клонирование репозитория**
```bash
git clone [https://github.com/newfpv/AiMovieSearcher.git](https://github.com/newfpv/AiMovieSearcher.git)
cd AiMovieSearcher
````

**2. Настройка окружения**
Переименуй файл `.env.example` в `.env` и заполни своими актуальными данными:

```bash
cp .env.example .env
nano .env
```

*Открой файл `.env` и укажи свои `TG_BOT_TOKEN`, `API_KEYS` (через запятую) и текст кнопок. Ты также можешь настроить `GROUP_THREAD_MAP`, чтобы ограничить работу бота конкретными чатами.*

**3. Запуск бота**
Убедись, что у тебя установлен Docker, затем выполни:

```bash
docker-compose up -d --build
```

## 🛠 Кастомизация

Ты можешь легко настроить поведение бота, отредактировав файл `.env`:

  - **Смена моделей:** Измени `MODEL_FALLBACK_LIST`. Полный список доступных моделей Google Gemini можно найти [здесь](https://ai.google.dev/gemini-api/docs/models).
  - **Настройка промпта:** Измени переменную `SYSTEM_PROMPT`, чтобы поменять язык ответа, настроить форматирование или добавить новые критерии к результатам поиска.
  - **Перевод кнопок:** Измени `TRAILER_BTN_TEXT` и `SOURCE_BTN_TEXT` под свой язык.

## ⚙️ Решение проблем

  - **ModuleNotFoundError: No module named 'dotenv':** Ты запустил бота со старым кэшем Docker. Принудительно пересобери образ с помощью `docker-compose build --no-cache`, а затем запусти `docker-compose up -d`.
  - **Бот игнорирует сообщения:** Проверь настройку `GROUP_THREAD_MAP` в файле `.env`. Убедись, что ID групп и веток (Thread IDs) указаны верно. Если оставить поле полностью пустым, бот будет отвечать всем.
  - **Ошибки API (429 RESOURCE\_EXHAUSTED):** Ты исчерпал дневные лимиты Google API. Добавь больше ключей в переменную `API_KEYS`. Бот будет автоматически их чередовать.
